HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • text headers are denoted using the "h1, h2, h3, h4, h5, h6" tags (each header gets smaller
  • a paragragh is denoted using the "p" tag
  • lists can be denoted as "ul" or unordered list, "ol" or ordered list, and "li" list item tags
  • an image is denoted using the "img" tag

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • each ruleset must be wrapped in curly braces "{}"
  • within each declaration, use a colon ":" to separate a property from it's values
  • within each ruleset, use a semicolon ";" to separate each declaration
  • colors can be denoted using common color names or hex color values

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stages a commit
  • git commit -m "(comment here)": commits changes with comment
  • git pull origin main: ensures that we have the latest commits (will return "Already up to date" if commits are up to date)
  • git push origin (branch name): pushes from branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • functions are independent units of behavior
  • objects are dynamic collections of properties that can be added, edited, or deleted
  • objects inherit from other objects called prototypes
  • functions are values that can be assigned to variables
  • functions can be defined inside other functions, so they can reference variables from the outer function
  • Control flow is the order in which a computer executes code in a script.
  • JS is a singlethreaded language (only one statement is executed at a time)
  • this single-threading is sometimes referred to as the stack
